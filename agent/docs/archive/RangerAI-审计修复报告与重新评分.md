## 二、P0 致命漏洞修复（3/3）

### SEC-P0-01：6 个 API 端点无认证（已修复）

**修复前**：任何人可通过公网无认证访问 `/api/chats`、`/api/tickets`、`/api/kols`、`/api/knowledge`、`/api/notifications`、`/api/audit-logs`，可读取全部业务数据，甚至可创建和删除数据。

**修复方案**：在 `http-router.mjs` 中添加统一的 JWT 认证中间件层。所有 `/api/*` 路由默认需要认证，仅白名单路由（`/api/auth/*`、`/api/stats`、`/api/health`、`/api/ws`、`/api/system/*`）豁免。

**验证结果**：

| 端点 | 修复前 | 修复后 |
|------|:------:|:------:|
| GET /api/chats | 200（泄露数据） | **401** |
| GET /api/tickets | 200（泄露数据） | **401** |
| GET /api/kols | 200（泄露联系方式） | **401** |
| GET /api/knowledge | 200（泄露知识库） | **401** |
| GET /api/notifications | 200 | **401** |
| GET /api/audit-logs | 200 | **401** |
| POST /api/chats | 200（可创建） | **401** |
| POST /api/tickets | 200（可创建） | **401** |
| DELETE /api/tickets/:id | 200（可删除） | **401** |

### SEC-P0-02：缺少 HSTS 头（已修复）

**修复前**：HTTP 响应中无 `Strict-Transport-Security` 头，存在 SSL 降级攻击风险。

**修复后**：API 和静态文件均返回 `Strict-Transport-Security: max-age=31536000; includeSubDomains`。

### SEC-P0-03：缺少 CSP 头（已修复）

**修复前**：无 Content-Security-Policy，存在 XSS 注入风险。

**修复后**：完整的 CSP 策略已部署，覆盖 `default-src`、`script-src`、`style-src`、`font-src`、`img-src`、`connect-src`、`frame-ancestors` 七个指令。

### SEC-P1-02：index.html 缓存控制（已修复）

**修复后**：`Cache-Control: no-cache, no-store, must-revalidate`，确保用户始终获取最新版本。

**修复后**：`/opt/rangerai-agent/` 和 `/opt/rangerai-web/` 目录权限已收紧，敏感配置文件设为 600。

### SEC-P1-05：安全头去重（已修复）

**修复前**：Caddy 和后端同时设置安全头，导致响应中出现重复头。
